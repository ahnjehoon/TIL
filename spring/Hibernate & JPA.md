
## 엔티티
- 데이터베이스 테이블과 매핑되는 객체
- 실제 데이터베이스에 저장되는 데이터를 표현하는 객체
- DTO와는 다르게 비지니스 로직을 포함할 수 있음
- 주요 특징
	- 식별자(Indentity)
	- 영속성(Persistence)
	- 테이블과 매핑됨
	- 기본 생성자 필수
	- final, enum interface, inner 클래스 사용 불가

## 엔티티 스냅샷
- Hibernate/JPA가 엔티티의 원본 상태를 보관하는 것
- 변경 감지(Dirty Checking)를 위해 사용
- 트랜잭션이 끝날때 엔티티와 스냅샷을 비교해서 변경된 부분 감지
- 처리 순서
	- 최초 조회 시점에 스냅샷 생성
	- 트랜잭션 커밋 시점에 아래 과정 수행
		- 스냅샷과 현재 엔티티 상태 비교
		- 변경감지
		- UPDATE 쿼리 생성 및 실행
	- 성능 최적화
